Code Challenge

Matrix rotation, QR factorization, and statistical analysis in a single workflow.

A technical solution for the Interseguro TI Division that connects a Go API (Fiber v3) with a Node.js API (Express 5) to perform matrix operations, Gram-Schmidt Modified QR decomposition, and global statistics across the resulting matrices.

How it works.

rotate_right

Matrix Rotation

7 rotation types supported.

Choose from clockwise (90°, 180°, 270°), transpose, horizontal flip, or vertical flip. The rotation is applied before QR factorization so the result reflects the transformed input.

grid_on

QR Factorization

Gram-Schmidt Modified algorithm.

The Go API decomposes the rotated matrix into an orthogonal matrix Q and an upper triangular matrix R. The algorithm handles any m×n matrix where rows ≥ columns and the input is not singular.

Analytics & Results.

Once factorization completes, the Node.js API processes the three output matrices and returns global statistics.

query_stats

Global Statistics

Maximum, minimum, average, sum, total elements, and number of matrices are computed across Q, R, and the rotated matrix in a single pass.

checklist

Diagonal Detection

Each square matrix is checked against a 1×10−10 tolerance to determine if it is diagonal. Results include the matrix name, index, and dimensions.

hub

Service Orchestration

The Go API calls the Node API with internal JWT auth. If the Node API is unavailable, the system degrades gracefully and returns Q, R, and rotated matrix without statistics.
